# 1. 整体架构设计

Hermes Agent 采用分层模块化架构设计，具有高扩展性、高可维护性和高兼容性，支持多种部署方式和扩展场景。

## 架构分层

Hermes Agent 从上到下分为四个核心层级，严格遵循单向依赖原则，上层依赖下层，下层不依赖上层：

| 层级 | 包含模块 | 核心职责 |
|------|----------|----------|
| **入口层** | `run_agent.py`、`cli.py`、`gateway/run.py`、`batch_runner.py` | 对外提供多种使用入口，包括 Python API、交互式 CLI、消息网关、批量执行工具。 |
| **核心逻辑层** | `AIAgent` 类（`run_agent.py`）、模型工具层（`model_tools.py`） | 实现对话核心循环、工具调用编排、错误处理、迭代控制，是整个系统的大脑。 |
| **基础能力层** | `agent/` 目录下所有模块（`prompt_builder.py`、`context_compressor.py`、`prompt_caching.py`等）、`tools/` 目录下所有工具实现 | 提供上下文管理、提示词构建、压缩、缓存、工具执行等通用基础能力。 |
| **存储层** | `hermes_state.py`、配置系统（`hermes_cli/config.py`） | 负责会话持久化、全文搜索、配置管理、数据持久化。 |

## 依赖关系

### 严格单向依赖链
```
入口层 ← 核心逻辑层 ← 基础能力层 ← 存储层
```
这种设计确保了各层的职责单一，修改下层不会影响上层，上层可以灵活替换下层的实现而不影响整体逻辑。

### 具体模块依赖
```
tools/registry.py (无任何依赖，所有工具的基础)
  ↑
tools/*.py (每个工具独立实现，调用 registry.register 注册自己)
  ↑
model_tools.py (导入 registry，触发工具发现，实现工具调度)
  ↑
run_agent.py、cli.py、batch_runner.py、gateway 等入口模块
```
工具注册中心是整个系统的最基础依赖，所有工具能力都基于注册中心实现，确保工具的可插拔性。

## 数据流走向

整个对话流程的数据流清晰，从用户输入到最终回复的完整路径：

```
1. 用户输入 → 入口层（CLI/Gateway/API）→ 核心层 AIAgent 实例
2. AIAgent 组装上下文：系统提示词 + 历史消息 + 技能指令 + 上下文文件 + 平台专属提示
3. 上下文预处理：添加缓存标记、Token 用量预估、压缩判断
4. LLM API 调用：适配对应的模型提供商 API，处理流式输出和回调通知
5. LLM 响应处理：
   a. 无工具调用：返回最终回复给用户，持久化会话到存储层，结束循环
   b. 有工具调用：进入工具处理流程
6. 工具执行：
   a. 校验工具是否在可用列表中
   b. 转换工具参数为正确的类型
   c. 判断是否可以并行执行
   d. 执行工具，捕获所有异常
   e. 格式化工具结果为 JSON 字符串
7. 工具结果追加到上下文，回到步骤 3，继续循环调用 LLM
```

## 核心设计理念

Hermes Agent 的架构设计遵循以下核心原则：

### 1. 无状态核心设计
AIAgent 实例本身没有全局状态，所有状态都保存在上下文和存储层中，支持多会话并发处理，方便横向扩展。

### 2. 可插拔工具系统
工具采用自动注册机制，新增工具只需要在 `tools/` 目录下添加独立的实现文件，无需修改核心代码，工具会在启动时自动被发现和注册。

### 3. 上下文自管理
上下文压缩、提示词缓存、Token 预算控制都由系统自动处理，用户不需要手动干预，最大化降低使用成本和复杂度。

### 4. 多平台原生兼容
同一核心逻辑支持多种平台使用，包括本地 CLI、各类消息平台（Telegram、Discord、Slack、企业微信等）、编辑器插件（VS Code、JetBrains等），不需要为不同平台重写核心逻辑。

### 5. 成本优先优化
架构设计优先考虑降低 Token 消耗和运行成本：
- 提示词缓存机制降低重复提示词的 Token 消耗
- 自动上下文压缩减少历史消息的 Token 占用
- 低成本辅助模型处理摘要、分类等非核心任务
- 工具执行结果自动截断，避免不必要的 Token 浪费

### 6. 安全第一原则
多层安全机制确保使用安全：
- 危险命令自动检测和审批
- 代码执行在隔离沙箱中运行
- 文件操作路径白名单限制
- 浏览器工具内网访问限制
- 多租户会话完全隔离

## 架构优势

### 1. 高扩展性
新增工具、技能、平台适配器都不需要修改核心代码，只需要添加对应的实现文件即可。

### 2. 高可维护性
模块化设计，每个模块职责单一，代码结构清晰，便于定位和修复问题。

### 3. 高兼容性
支持多种模型提供商、多种部署方式、多种使用场景，适应性强。

### 4. 高性能
通过缓存、压缩、并行执行等优化，在保证功能的同时最大化性能，降低资源消耗。

### 5. 高稳定性
完善的错误处理和重试机制，核心逻辑异常不会导致整个系统崩溃，会话数据不会丢失。
